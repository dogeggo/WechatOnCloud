# 当前分支与 upstream/main 功能对比

更新时间：2026-06-15

对比基线：

- 当前分支：`main` / `origin/main`，提交 `11cf383`
- 上游分支：`upstream/main`，提交 `f5e5bb5`，标签 `v1.2.1`
- 共同祖先：`13568a8`
- 差异规模：当前分支独有 12 个提交，`upstream/main` 独有 27 个提交

## 背景说明

当前分支是基于 `upstream/main` 的定制产品分支，已经在部署方式、服务端结构、认证会话、前端模块化和桌面交互上做过较大改造。

`upstream/main` 在分叉后继续新增了多应用平台、实例图标、日志诊断、版本检测、桌面输入模式和若干 Docker/看门狗修复。这些功能大多仍落在上游原有的面板结构中，后续提取到当前分支时建议按功能小批量移植，不建议直接整体合并。

## 当前分支功能

### 部署与运行定制

- 更新 `docker-compose.yml` 与面板构建配置，适配当前定制产品部署方式。
- 默认镜像源从 GHCR 调整为 Docker Hub，并修复容器权限相关配置。
- 新增 `nginx.wechat.dogegg.online.conf`，提供当前产品域名/反代部署配置。
- 移除 GPU 硬件编码相关配置，改为固定软件渲染，降低宿主机图形能力依赖。
- 裁剪上游通用发布资产和飞牛相关目录，例如 CI 工作流、Telegram 通知脚本、`fnos` 目录和部分根目录文档。

### 服务端架构与安全

- 将服务端从单体入口拆分为领域目录：
  - `auth`：认证配置、认证管理、会话管理。
  - `config`：面板配置读取与归一化。
  - `desktop`：桌面/VNC 反代相关逻辑。
  - `docker`：Docker 实例控制封装。
  - `http`：Host 校验、请求安全、请求工具、上传工具、速率限制和 HTTP 错误封装。
  - `instance`：实例管理、实例控制、资源保护和实例存储。
  - `watchdog`：运行状态/看门狗管理。
- 增强会话管理和持久化能力，简化运行配置。
- 增加 Host Guard、请求安全、上传处理和速率限制等 HTTP 边界能力。
- 将实例创建、删除、重命名、安全校验、资源限制等逻辑从路由入口中拆出，便于后续维护。

### 前端结构与管理体验

- 将前端从页面级大文件拆分为更细的领域模块：
  - `domain`：设备、实例、内存限制、数据卷路径等领域规则。
  - `features/admin`：实例创建、删除、重命名、安全、数据卷管理和管理面板加载逻辑。
  - `features/desktop`：桌面 iframe、VNC、剪贴板、输入条、文件上传、运行时操作等逻辑。
  - `features/instances`：实例列表加载逻辑。
  - `components` 和 `utils`：空状态、图标、错误处理、格式化等复用代码。
- 优化管理页和桌面页 UI，配合当前定制产品的登录、实例管理和桌面控制流程。

### 桌面/VNC 与输入体验

- 增加 VNC 常驻保活逻辑，切换页面时尽量保持实例连接不断开。
- 优化中文输入条，支持“独立输入”和“发送”分离操作。
- 补强桌面控制权校验，避免无控制权时错误发送输入。
- 优化剪贴板粘贴可靠性，并在输入前增加焦点校验。
- 优化 VNC 音频和桌面 frame 相关前端逻辑。

## upstream/main 功能

### 多应用平台能力

- 引入实例 `appType` 数据模型基础，实例不再只按“微信”硬编码处理。
- 创建实例时支持按应用类型选择；当前支持微信、QQ 与 Chromium 浏览器。
- 增加 `app-ctl.sh`、`app-defs.sh`、`woc-app-init.sh` 等应用抽象脚本。
- 安装状态链路改为 `appStatus` / `app-ctl <appType>`，不再暴露微信命名接口。
- UI 文案和实例状态按应用类型泛化，减少前端到处写死“微信”。

### 实例图标能力

- 实例头像由首字母改为按应用类型展示图标。
- 支持内置精选图标。
- 支持上传并裁剪自定义实例图标。
- 新增 `panel/web/src/AppIcon.tsx` 承载实例图标展示逻辑。

### 管理面板与可观测性

- 实例“管理”菜单改为悬浮图层展开。
- 新增持久化实例日志，容器重建后仍可保留实例日志。
- 新增全局日志系统和一键诊断包导出。
- 诊断/关于区域改为扁平区块样式，并修复按钮断行和 `chip` 类缺失问题。
- 显示构建版本号。
- 自动检测 Docker Hub/GHCR 新版本，并在 UI 中显示红点提醒。
- 修复开发版构建被误标记为“已是最新”的版本展示问题。

### 桌面输入与控制修复

- 新增输入法双模式切换：无感输入 / 转发输入。
- 深度修复无感输入丢字问题。
- 修复输入模式切换不生效和切换后页面卡死问题，改为整页重载处理。
- 修复 `typeInInstance` 卡顿约 2 秒的问题，根因是 `xclip` 文件描述符未重定向。

### Docker、看门狗与运维改进

- 修复 Docker 脚本权限，补齐 `00-woc-identity` 的显式 `chmod 755`。
- Chromium 启动参数增加 `--disable-background-networking`，关闭后台 phone-home 请求。
- VNC 响应性探测默认关闭，可通过环境变量重新启用。
- README 写入浏览器实例支持并做精简整理。
- GitHub Issue 新回复也会触发 Telegram 简短通知。

## 重叠与差异点

- 输入能力：两个分支都改过输入体验。当前分支偏向中文输入条、剪贴板和控制权校验；`upstream/main` 偏向输入模式切换、无感输入修复和 `xclip` 卡顿修复。后续合入时应逐项比较，不建议直接覆盖当前 `features/desktop` 实现。
- VNC 能力：当前分支已有 VNC 常驻保活；上游新增的是 VNC 响应性探测默认关闭和环境变量开关，二者可以互补。
- 部署能力：当前分支已经切到 Docker Hub 并裁剪通用发布流程；上游新增版本检测时同时考虑 Docker Hub/GHCR，后续提取时需要按当前镜像源策略调整。
- 面板架构：当前分支已经把服务端和前端拆成领域模块；上游新增功能主要改在 `panel/server/src/index.ts`、`panel/server/src/docker.ts`、`panel/server/src/store.ts`、`panel/web/src/pages/Admin.tsx`、`panel/web/src/pages/Desktop.tsx` 等较集中的旧结构文件中。提取时需要手工适配到当前模块化目录。
- 多应用能力：上游的 `appType`、应用脚本和图标能力是较完整的一组功能，和当前实例管理、Docker 控制、前端领域模型都会交叉，属于高耦合迁移。

## 建议提取顺序

1. 先提取低耦合修复：Docker 权限、Chromium 后台网络参数、VNC 探测默认关闭。
2. 再提取可观测性能力：持久化日志、全局日志、一键诊断包。
3. 再提取版本展示和新版检测，按当前 Docker Hub 镜像策略调整。
4. 对输入能力做差异审查，只移植当前分支缺失的 bugfix。
5. 最后评估是否要引入多应用平台和实例图标，因为这会影响实例数据模型、Docker 控制脚本、服务端 API 和前端管理流程。

## 注意事项

- 不建议直接 `merge upstream/main`，当前分支和上游在面板架构、文件组织和产品裁剪上已经明显分叉。
- 后续选择某个上游功能时，优先使用“提交 + 相关文件 + 当前模块适配”的方式提取。
- 每次提取后至少保证编译通过，并检查是否误恢复当前分支已裁剪的 CI、README、飞牛目录或发布流程。
